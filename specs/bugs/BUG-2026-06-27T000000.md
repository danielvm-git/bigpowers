# BUG-2026-06-27T000000: 8 SKILL.md files exceed line cap — CI blocked

## Problem

The CI workflow (`.github/workflows/sync-skills.yml`) runs `bash scripts/validate-doctrine.sh`, which calls `scripts/check-skill-size.sh`. This lint enforces tiered line caps on SKILL.md files:

- **Critical-path skills:** 150 lines max (develop-tdd, migrate-spec, release-branch)
- **Utility skills:** 120 lines max (deploy, publish-package, smoke-test, validate-contracts, wire-ci)

**8 files currently exceed their caps:**

| File | Lines | Cap | Over by |
|------|-------|-----|---------|
| wire-ci/SKILL.md | 308 | 120 | 188 |
| migrate-spec/SKILL.md | 298 | 150 | 148 |
| publish-package/SKILL.md | 261 | 120 | 141 |
| smoke-test/SKILL.md | 191 | 120 | 71 |
| validate-contracts/SKILL.md | 182 | 120 | 62 |
| release-branch/SKILL.md | 189 | 150 | 39 |
| deploy/SKILL.md | 165 | 120 | 45 |
| develop-tdd/SKILL.md | 175 | 150 | 25 |

**CI impact:** Every push to `main` runs `validate-doctrine.sh` → `check-skill-size.sh` exits 1 → CI fails on the Sync Skills workflow. This blocks all future releases until fixed.

## 4-Phase Root Cause Analysis

### Phase 1 — Reproduce

```bash
bash scripts/check-skill-size.sh
```

Expected: exit 0 (all within caps). Actual: exit 1 with 8 files listed.

### Phase 2 — Isolate

The lint rule is at `scripts/check-skill-size.sh`. The cap values (150/120) were chosen to give a reasonable upper bound. The 8 files grew beyond their caps incrementally as features and edge-case documentation were added inline to SKILL.md instead of being moved to REFERENCE.md.

### Phase 3 — Hypothesize

| # | Hypothesis | Falsification |
|---|------------|---------------|
| H1 | SKILL.md contains procedural detail that belongs in REFERENCE.md | Read each file; confirm blocks that describe "how" (scripts, examples, edge cases) vs "what" (workflow steps, interface) |
| H2 | Cap values are too low for real-world content | Caps were chosen based on principle (Ousterhout information-hiding), not empirical data. 120 lines for a complex utility skill may be too tight. |
| H3 | REFERENCE.md files exist but aren't being used | Check which skills have REFERENCE.md; if they exist but SKILL.md is still over cap, content wasn't migrated |

### Phase 4 — Verify

Reading the 8 files confirms H1: all contain inline procedural detail (full command blocks, extended examples, edge-case handling tables, multi-language package registry URLs) that belongs in REFERENCE.md. Some files (deploy, publish-package, smoke-test, validate-contracts, wire-ci) have no REFERENCE.md at all.

**Root cause:** Skills grew over time with features/edge-cases documented inline in SKILL.md rather than moved to REFERENCE.md. The lint cap existed in prose but was only recently enforced (via validate-doctrine.sh), catching the accumulated drift.

## Resolution

**Fixed:** 2026-06-26
**Root cause confirmed:** Skills grew over time with procedural detail documented inline in SKILL.md rather than in REFERENCE.md. The lint cap existed but wasn't enforced until validate-doctrine.sh was wired into CI.
**Fix applied:** Moved long code blocks, procedural examples, reference tables, optional post-migration steps, and configuration details from SKILL.md to REFERENCE.md for all 8 over-length files.
**Hardening added:** `scripts/check-skill-size.sh` now runs as part of CI (via validate-doctrine.sh) and scripts/sync-skills.sh. Any future SKILL.md that exceeds its cap will block CI at commit time.
**Evidence:** `bash scripts/check-skill-size.sh` exits 0; `bash scripts/validate-doctrine.sh` all checks pass; `npm run compliance` 97% SCORE GATE PASS.
