# BUG-2026-06-02T164500: CI Sync Skills + Release failures

## Problem

After push `bc1cf20` (`chore(specs): archive legacy spikes and align YAML paths`), GitHub Actions failed:

- **Sync Skills on Push** (run `26833989310`): `git diff --exit-code` on generated artifacts
- **Release** (run `26833989292`): `npm ci` lockfile mismatch

**Expected:** Both workflows green on `main`.

**Reproduce:**

```bash
gh run list --limit 3
gh run view 26833989310 --log-failed
gh run view 26833989292 --log-failed
```

## Root Cause Analysis

Two independent defects:

1. **sync-skills.sh BSD sed bug** — `sed 's/[[:space:]]\+/ /g'` on macOS treats `\+` as a literal `+`, stripping `+` from descriptions (e.g. `release-plan.yaml + epic` → `yaml  epic`). CI regenerates artifacts with correct `+`, failing the committed stale `.mdc`/Gemini files.

2. **sync-skills.sh version extraction** — `grep '"version":' package.json` also matched `scripts.version`, corrupting `gemini-extension.json` and breaking manifest metadata.

3. **package-lock.json drift** — Lockfile still at `1.1.1` while `package.json` is `1.3.0` after semantic-release; `npm ci` in Release workflow failed.

**Risk level:** High (blocks release and sync gates on every push)

## TDD Fix Plan

1. **RED**: CI log shows artifact diff on `trace-requirement.mdc`
