# BUG-2026-06-11T120000: TUI markup tags render as literal text; layout diverges from prototype

## Problem

**Actual behavior:** The TUI dashboard shows raw blessed markup tags as literal text in the terminal (e.g. `{bold}{cyan}Pipeline{/cyan}{/bold}`, `{dim}active_flow:{/dim}`, `{yellow}BCP/hr: 1.40{/yellow}`). The screen appears as a black terminal with unformatted, tag-polluted text.

**Expected behavior:** All blessed markup tags should be interpreted as styling instructions — bold, colors, and dim should render visually. The layout should approximate the web prototype: a title bar, 6-cell metrics row, horizontal pipeline strip, three-panel middle section (EPIC QUEUE | CURRENT ACTION | FILE SYSTEM), labeled state.yaml grid, and ledger table.

**Prototype reference:** The web prototype shows:
1. Title bar: "bigpowers factory v2 — seed → epics → mvp"
2. 6 metric cards in a row: epics, stories, BCPs delivered, avg cycle, BCP/hr, semver
3. Horizontal pipeline with numbered steps and `›` arrows
4. Three side-by-side panels with ALL-CAPS headers
5. SPECS/STATE.YAML section with labeled rectangular key-value cells
6. CYCLE TIME LEDGER at the bottom

## Root Cause Analysis

**Primary bug:** Every `blessed.box()` in the TUI layout module is instantiated **without `tags: true`**. The blessed library requires this option to parse and render markup like `{bold}`, `{cyan}`, `{dim}`, etc. Without it, `setContent()` treats those strings as literal characters, so the user sees raw tag text instead of styled output. This affects all 6 boxes: metricsBar, pipeline, epicQueue, actionLog, fsPanel, and ledger.

**Secondary gaps (layout, not crashes):**
- Pipeline renders as a vertical numbered list; prototype shows a horizontal strip with `›` separators and a step counter (`step N / 13`)
- Metrics bar is one concatenated line; prototype has 6 individual labeled cards
- State-yaml section renders key-value pairs as plain indented text; prototype uses discrete labeled boxes
- No project title bar exists in the TUI
- Panel headers are provided by the blessed border only; prototype uses ALL-CAPS section labels

Risk level: Low — all changes are cosmetic/rendering only. No data layer is affected.
