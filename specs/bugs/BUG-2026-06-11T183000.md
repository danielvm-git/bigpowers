# BUG-2026-06-11T183000: dashboard CLI not exposed as a bin entry — only works inside bigpowers repo

## Problem

Running `npm run dashboard` in any project other than bigpowers itself fails with:

```
npm error Missing script: "dashboard"
```

The dashboard is expected to work from any project directory (reading `specs/` from the current working directory). Instead it only works inside the bigpowers repository because `dashboard` is listed only as an npm `scripts` entry, not a published `bin` entry.

**Actual behavior:** `npm run dashboard` in a foreign project → "Missing script: dashboard".  
**Expected behavior:** Users who install bigpowers globally can run `bigpowers-dashboard` (or `npx bigpowers-dashboard`) from any project directory.

## 4-Phase Root Cause Analysis

### Phase 1 — Reproduce

**Environment:** bigpowers@2.33.0 installed globally in foreign project directory.

**Steps to reproduce:**
1. `cd /tmp && mkdir test-dashboard && cd test-dashboard`
2. `npm install -g bigpowers` (or use an existing global install)
3. `bigpowers-dashboard --help`

**Expected:** Help text displayed.
**Actual:** `zsh: command not found: bigpowers-dashboard`

**Captured output (2026-06-26):**
```
$ which bigpowers-dashboard
bigpowers-dashboard not found
$ npm ls -g bigpowers 2>/dev/null | grep bigpowers
`-- bigpowers@2.33.0
$ which bigpowers
/usr/local/bin/bigpowers
```

Only `bigpowers` is linked. `bigpowers-dashboard` does not exist on PATH.

### Phase 2 — Isolate

**Narrow to module:** `package.json` `bin` field.

```json
"bin": {
  "bigpowers": "./bin/bigpowers.js"
}
```

Only one entry. `scripts.dashboard` exists but `scripts` are not published to consumers. The dashboard binary (`dashboard/bin/dashboard.js`) is a proper CLI with `#!/usr/bin/env node`, full arg parsing, `--help`, `--web`, `--port`, `--project`. It is **bin-ready** but lacks the declaration.

**Binary-search in git:**
```
git log --oneline --all -- package.json | head -5
9ed35f2 chore(release): 2.33.0 [skip ci]
19a25d3 chore(release): 2.32.0 [skip ci]
...
```
The `bin` field has only ever had `bigpowers` — never `bigpowers-dashboard`. The dashboard was added in `dashboard/` alongside the scripts entry, but the `bin` registration was simply never written.

### Phase 3 — Hypothesize

| # | Hypothesis | Falsification Test |
|---|------------|-------------------|
| H1 | `bin` field in `package.json` is missing the `bigpowers-dashboard` entry | Add `"bigpowers-dashboard": "./dashboard/bin/dashboard.js"` to `bin`, run `npm pack && npm install -g` → `bigpowers-dashboard --help` succeeds |
| H2 | Dashboard binary (`dashboard/bin/dashboard.js`) is not executable or lacks proper shebang | `head -1 dashboard/bin/dashboard.js` shows `#!/usr/bin/env node`; file has `-rwxr-xr-x` — correct. Falsified. |
| H3 | Dashboard binary has internal dependency that fails outside bigpowers repo | Binary uses only `require('path')`, `require('../src/web/server')`, `require('../src/tui/index')` — all resolved relative to `dashboard/bin/` → `dashboard/src/`, contained within package. Falsified by inspection. |
| H4 | `scripts.dashboard` in package.json is the intended mechanism (works for consumers who `npm explore`) | `scripts` entries are not exported by npm. `npm explore bigpowers -- npm run dashboard` would work, but this is undocumented and cumbersome. The standard npm mechanism is `bin`. |

### Phase 4 — Verify

**Falsification of H1 (the root cause):**
1. Temporarily add `"bigpowers-dashboard": "./dashboard/bin/dashboard.js"` to `package.json` `bin` field.
2. `npm pack` → creates `bigpowers-2.33.0.tgz`
3. `mkdir -p /tmp/bp-test && cd /tmp/bp-test && npm install -g ../bigpowers-2.33.0.tgz`
4. `bigpowers-dashboard --help` → help text displayed, exit 0

**Result:** H1 confirmed. The sole root cause is the missing `bin` entry. No logic changes needed anywhere.

**Verdict:** Verified single root cause — missing `"bigpowers-dashboard": "./dashboard/bin/dashboard.js"` in `package.json` `bin` field.

**Risk level:** Low — additive change, zero breaking impact on existing users.
