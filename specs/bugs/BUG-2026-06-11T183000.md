# BUG-2026-06-11T183000: dashboard CLI not exposed as a bin entry â€” only works inside bigpowers repo

## Problem

Running `npm run dashboard` in any project other than bigpowers itself fails with:

```
npm error Missing script: "dashboard"
```

The dashboard is expected to work from any project directory (reading `specs/` from the current working directory). Instead it only works inside the bigpowers repository because `dashboard` is listed only as an npm `scripts` entry, not a published `bin` entry.

**Actual behavior:** `npm run dashboard` in a foreign project â†’ "Missing script: dashboard".  
**Expected behavior:** Users who install bigpowers globally can run `bigpowers-dashboard` (or `npx bigpowers-dashboard`) from any project directory.

## Root Cause Analysis

The `package.json` `bin` field registers only one executable:

```json
"bin": {
  "bigpowers": "./bin/bigpowers.js"
}
```

`dashboard/bin/dashboard.js` is never registered in `bin`. When `npm install -g bigpowers` runs, npm links only `bigpowers` onto `$PATH`. The `dashboard` key under `scripts` is a developer convenience for working within the bigpowers repo; it is not surfaced to consumers.

The dashboard binary itself (`dashboard/bin/dashboard.js`) is already portable â€” it defaults `projectRoot` to `process.cwd()` and accepts `--project` to override, so no logic changes are needed to the dashboard code. Only the `bin` registration is missing.

**Risk level:** Low â€” additive change, no breaking impact on existing users.
