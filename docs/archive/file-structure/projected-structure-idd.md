# Projected ArcBlock/idd Project Directory Structure

Below is the projected filesystem folder and file structure for a hypothetical project built using **ArcBlock/idd** (Intent-Driven Development).

```text
my-project/
├── .github/                       # CI/CD workflows and action configurations
├── intent/                        # Project-level Intent (vision & module indexes)
│   ├── INTENT.md                  # Project entrypoint (anchor vision, chamber registry)
│   ├── decisions.md               # Log of design & product choices from interview loops
│   ├── _archive/                  # Deprecated or superseded intent files
│   ├── _data/                     # Autogenerated intent status metrics (read-only)
│   │
│   ├── records/                   # Append-only chronological meeting and discussion logs
│   │   ├── INDEX.md               # Log index summary
│   │   └── interview-2026-06-11.md
│   │
│   ├── architecture/              # Global structural limitations and boundaries
│   │   ├── DEPENDENCIES.md        # Graph detailing import boundaries between modules
│   │   └── BOUNDARIES.md          # Architectural boundary rules (size, lines budgets)
│   │
│   └── specs/                     # General system structure specifications
│
├── src/                           # Application Source Code (with local chamber intents)
│   ├── core/                      # Main module
│   │   ├── intent/                # Module-level intent
│   │   │   ├── INTENT.md          # Anchor, module responsibilities, API signatures
│   │   │   └── decisions.md       # Module-specific decisions
│   │   └── index.ts
│   │
│   └── chambers/                  # Independent execution scopes (chambers)
│       └── terminal/
│           ├── intent/
│           │   └── INTENT.md      # Terminal chamber intent and constraints
│           └── cli.ts
│
├── tests/                         # Automated tests (verifiers)
│   └── integration/
│
├── CLAUDE.md                      # Agent fast-reference instructions (commands, conventions)
└── package.json                   # Project manifest (includes versioning and dev dependencies)
```
